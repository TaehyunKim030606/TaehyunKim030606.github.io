# LinkedIn 프로필 기본 정보 입력 기록

오늘은 개발자 포트폴리오를 준비하는 과정의 하나로 LinkedIn 프로필에 기본 정보를 입력했다.

단순히 SNS 계정을 만든 것이 아니라, 앞으로 내가 진행하는 프로젝트와 학습 기록, GitHub 블로그를 외부에 연결하기 위한 기반을 만드는 작업이었다.

## 왜 LinkedIn을 정리했는가?

개발자로 성장하기 위해서는 공부한 내용과 프로젝트 결과물을 단순히 개인적으로 보관하는 것에서 끝나면 안 된다고 생각했다.

내가 어떤 기술을 공부하고 있는지, 어떤 프로젝트를 진행했는지, 어떤 문제에 관심이 있는지를 외부에서 확인할 수 있도록 정리하는 과정이 필요하다.

GitHub 블로그가 내가 공부하고 만든 것을 자세히 기록하는 공간이라면, LinkedIn은 그 기록을 외부 사람들에게 보여주는 프로필 역할을 한다.

즉, 두 플랫폼의 역할은 다음과 같이 나눌 수 있다.

* GitHub Blog: 학습 과정, 프로젝트 회고, 개발 기록을 자세히 정리하는 공간
* GitHub Repository: 실제 코드와 프로젝트 결과물을 관리하는 공간
* LinkedIn: 나의 프로필, 관심 분야, 활동 이력을 보여주는 외부 연결 창구

## 오늘 입력한 기본 정보

오늘 LinkedIn에는 기본적인 프로필 정보를 먼저 입력했다.

아직 완성된 포트폴리오는 아니지만, 앞으로 채워나갈 수 있는 기본 틀을 만든 것에 의미가 있다.

입력한 정보는 다음과 같다.

* 이름
* 학교 및 전공
* 관심 분야
* 프로젝트 경험
* GitHub 블로그와 연결할 수 있는 기본 프로필 정보

특히 개발자로서의 방향성을 보여주기 위해 단순한 자기소개보다는 앞으로 기록할 프로젝트와 학습 방향을 중심으로 정리하려고 했다.

## 오늘 작업하면서 느낀 점

오늘 LinkedIn 기본 정보를 입력하면서 포트폴리오는 한 번에 완성하는 것이 아니라는 생각이 들었다.

처음부터 완벽한 경력과 프로젝트가 있어야 하는 것이 아니라, 지금 내가 배우고 있는 것과 진행 중인 것을 꾸준히 기록하는 것이 중요하다.

특히 개발자는 결과물도 중요하지만, 그 결과물을 만들기까지의 과정도 중요하다.

앞으로 GitHub 블로그에는 다음과 같은 내용을 꾸준히 기록할 예정이다.

* GitHub Pages와 Jekyll 블로그 제작 과정
* 다양한 프로젝트 진행 과정
* Raspberry Pi 기반 네트워크 모니터링 연구
* 개발 공부 기록

오늘 LinkedIn 프로필을 정리한 것은 작은 작업이지만, 앞으로 나의 개발 활동을 외부에 보여주기 위한 첫 단계라고 생각한다.
