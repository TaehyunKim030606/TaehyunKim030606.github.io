오늘은 GitHub Pages와 Jekyll을 이용해서 나만의 개발 블로그를 만들었다.

## 왜 블로그를 만들었나

앞으로 소프트웨어공학과 학생으로서 공부한 내용, 프로젝트 경험, 창업 아이디어, 연구실 활동을 꾸준히 기록하기 위해 블로그를 시작했다.

단순히 결과만 남기는 것이 아니라, 내가 어떤 문제를 만났고 어떻게 해결했는지를 기록하면서 성장 과정을 남기고 싶다.

## 오늘 배운 개념

오늘 배운 핵심 개념은 다음과 같다.

- GitHub Pages는 GitHub 저장소를 웹사이트로 배포해주는 기능이다.
- Jekyll은 Markdown 파일을 HTML 웹페이지로 변환해주는 정적 사이트 생성기다.
- Chirpy는 Jekyll 기반의 블로그 테마이다.
- GitHub Actions는 Jekyll 빌드와 배포를 자동으로 실행해준다.

## 앞으로 기록할 내용

앞으로 이 블로그에는 다음과 같은 내용을 정리할 예정이다.

- Python, Git, 데이터베이스, 네트워크 공부 기록
- Raspberry Pi 기반 네트워크 연구
- 창업 아이디어와 비즈니스 모델
- 해커톤 및 팀 프로젝트 회고
- 개발자로 성장하는 과정

## 느낀 점

처음에는 GitHub Pages, Jekyll, Chirpy, GitHub Actions의 관계가 헷갈렸지만, 구조를 이해하고 나니 생각보다 명확했다.

앞으로 이 블로그를 단순한 기록장이 아니라, 나의 성장 포트폴리오로 만들어가고 싶다.
